# AI 日志主说明

用于记录 AI 参与的代码/文档修改，便于后续迭代、回溯与复盘。

## 目录结构

- `ai_logs\AI_LOGS_MAIN.md`：规范说明（本文件）
- `ai_logs\index.md`：总索引（最新在上）
- `ai_logs\change\YYYY-MM\*.md`：已落地或将落地的工程变更
- `ai_logs\analysis\YYYY-MM\*.md`：未落地的分析、设计、讨论与建议

## 记录分类

- 更改：仅记录已实际发生或即将落地的修改（代码、配置、数据、基础设施）
- 分析：仅记录未落地的思考与结论（调研、定位、设计、选型、风险判断）

## 统一元信息（更改 / 分析都要写）

- 来源类型（必填）：提问 / 处理
- 来源说明（必填）：一句话说明来自哪一次提问或哪一次处理

---

## 更改记录要求

类型不再使用英文缩写，改为中文二级分类：

- 更改类型-动作（必填，可多选）：新增功能 / 修复问题 / 结构重构 / 性能优化 / 实验流程变更 / 环境部署变更 / 数据处理变更
- 更改类型-范围（必填，单选）：局部模块 / 跨模块 / 全局架构

另外必须包含：

- 变更状态（必填）：草稿 / 已应用 / 已验证 / 已稳定
- 需求/目标
- 变更文件
- 变更摘要（变更内容 / 变更原因 / 影响范围）
- 后续迭代建议（可不填）
- 修改注意事项（可不填）

---

## 分析记录要求

为避免“多选过多”，分析类型改为二级且默认单选：

- 分析类型-目的（必填，单选）：问题定位 / 方案设计 / 方案选型 / 对比评估 / 风险评审 / 结果复盘
- 分析类型-产出（必填，单选）：事实结论 / 推理判断 / 决策建议

另外必须包含：

- 结论状态（必填）：初步 / 可用 / 已验证 / 稳定
- 输入内容
- 分析结果
- 关键结论
- 启发或下一步工作（可不填）
- 补充说明（风险点 / 适用范围 / 限制条件，可不填）

---

## 记录要求

- 采用倒序（最新在上）
- 编号递增并保证唯一性
- 两条记录之间使用 `---` 分隔
- 减少冗余，突出核心内容
- 关键结论必须自包含：不依赖上下文、不使用模糊指代、方案需完整列出

---
