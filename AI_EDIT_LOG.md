# AI 修改记录日志

> 用于记录 AI 参与的代码/文档修改，便于后续迭代、回溯与复盘。

## 记录规范（增强版）

### 1. 记录分类

- 更改：仅记录“已实际发生或即将落地”的代码 / 配置 / 数据 / 基础设施修改
  
- 分析：所有未落地的讨论、设计、推荐、解释、文档内容

---

### 2. 更改记录要求

必须包含：

- 变更类型（必填，可多选）：新增功能（feat）、修复问题（fix）、结构重构（refactor）、性能优化（perf）、实验配置或训练流程变更（experiment）、环境或工具链或部署变更（infra）、数据处理或特征工程变更（data）
- 变更类型（必填）：草稿 / 已应用 / 已验证 / 已稳定
- 变更内容：（描述修改的具体内容，如“修复了某个函数的边界条件错误”、“优化了某个算法的时间复杂度”、“添加了一个新的 API 接口”等）
- 变更原因：（说明为什么需要进行该修改，如“修复了一个已知的 bug”、“提升了性能”、“满足了新的需求”等）
- 影响范围：（评估该修改对项目的潜在影响范围，如“仅影响文档内容”、“影响`XXX`文件、或是`XXX`模块的功能”、“可能引入新的依赖”、“改变了文件结构，需要考虑import的问题”等）
- 后续迭代建议（可选）：（如果有风险或待办事项，请在此处写入，如“需要在后续版本中进行回归测试”、“需要在后续版本中优化相关功能”、“需要在后续版本中添加相关文档说明”等。待办事项使用，`- [ ]`表示代办，完成后使用`- [x]`。
- 修改注意事项（可选）：（如果有其他需要注意的事项，请在此处写入，如“该修改涉及到敏感代码，请谨慎评审”、“该修改可能引入新的依赖，请确保相关环境配置正确”等）

---

### 3. 分析记录要求（增强）

必须包含：

- 分析类型（必填，可多选）：调研 / 理解 / 设计 / 选型 / 对比 / 验证 / 优化 / 风险
- 结论状态（必填）：初步 / 推导 / 可用 / 已验证 / 稳定
- 输入内容：（简要描述本次对话的输入内容，如“用户询问了某个功能的使用方法”、“用户提供了某段代码进行分析”、“用户提出了某个需求进行讨论”等）
- 分析结果：（简要描述本次对话的分析结果，如“提供了该功能的使用方法说明”、“分析了该段代码的逻辑并指出了潜在问题”、“讨论了该需求的可行性并提出了实现方案”等）
- 关键结论。
- 启发，或是下一步可以完成的工作：使用Markdown 任务列表语法。
- 补充说明。

---

### 5. 记录顺序

- 采用倒序（最新在上），如果有冲突的，优先考虑最新的，但也要考虑最新版记录有误的可能性。
- 编号递增（保证唯一性）
- 两条记录之间使用“---”分隔

---

### 6. 目标原则

- 可追溯：能还原分析过程；
- 可审计：区分分析与实际修改；
- 可复用：能指导实验与开发；

---





## 参考格式

```
### 更改 N

时间：（时间戳，如 2026-03-24 15:23:00 +08:00）

变更类型：（feat / fix / refactor / perf / experiment / infra / data）（说明：本次修改属于哪一类工程变更）

变更状态：（草稿 / 已应用 / 已验证 / 已稳定）（说明：当前修改的落地与验证状态）

需求/目标：（本次修改要解决的问题或实现的目标）

变更文件：（涉及的文件，文件夹路径，如 src/model/xxx.py）（说明：可以多个文件，文件夹）

变更摘要：
- 变更内容：（具体修改了什么，如“新增某模块 / 修改某函数逻辑”）
- 变更原因：（为什么要改，如“修复 bug / 提升性能 / 支持新功能”）
- 影响范围：（影响哪些模块或功能，如“仅影响训练流程 / 影响全局推理”）

后续迭代建议。

修改注意事项：（风险或注意点，如“可能影响兼容性 / 依赖特定环境”）
```

```
### 分析 N
时间：（时间戳，如 2026-03-24 15:23:00 +08:00）

分析类型：（调研 / 理解 / 设计 / 选型 / 对比 / 验证 / 优化 / 风险，可多选）（说明：本次分析属于哪一类思考活动）

结论状态：（初步 / 推导 / 可用 / 已验证 / 稳定）（说明：当前结论的可靠程度）

输入内容：（用户问题 / 提供的信息 / 分析背景）

分析结果：（详细分析过程，可分点描述逻辑推导或信息整理）

关键结论：（最终得到的核心结论，用于后续决策或复用）

启发，或是下一步可以完成的工作：（使用Markdown 任务列表语法）

补充说明（可选）：
- 风险点：（潜在问题或不确定性）
- 适用范围：（该结论适用于哪些场景）
- 限制条件：（在哪些情况下不成立）
```
