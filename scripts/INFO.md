# scripts/INFO.md

## 目录职责

`scripts/` 存放项目脚本，包含运行入口脚本与各类辅助工具脚本。

## 关键内容

| 文件/目录 | 说明 |
| --- | --- |
| `check_deps.py` | 检查依赖环境 |
| `clear_cache.py` | 清理缓存与中间产物 |
| `inspect_config.py` | 检查配置解析结果 |
| `inspect_dataset.py` | 检查数据集加载和切分 |
| `rebuild_dtw_cache.py` | 重建 DTW 相关缓存 |
| `smoke_test.py` | 命令行单批次冒烟检查 |
| `run/` | 核心运行入口脚本（训练/数据处理/调参/续训） |
| `experiments/` | 不走主 pipeline 的独立实验脚本 |

## 输入/输出

- **输入**：项目配置、数据集、已生成缓存/输出。  
- **输出**：终端检查结果、缓存文件更新或独立实验产物。

## 调用关系

- 该目录脚本通常由开发者手动执行，不属于训练主流程的必经步骤。  
- 多数脚本会调用 `libcity` 内部模块完成检查、训练或复用通用逻辑。

## 修改注意事项

1. 优先保持脚本“可独立执行”，避免隐藏依赖。  
2. 涉及删除行为（如清理缓存）时要限制作用范围，避免误删输出目录。  
3. `run/` 目录参数变更需同步 Web 端调用逻辑。  
4. 新增脚本后建议在根目录 `INFO.md` 补充用途说明。
